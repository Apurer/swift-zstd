# Security Policy

## Supported Versions
The project follows Semantic Versioning. The latest release and the current default branch receive security fixes when feasible.

## Reporting a Vulnerability
- Use GitHub's Private Vulnerability Reporting to notify the maintainers privately.
- Please include steps to reproduce, affected versions, and any known mitigations.

## Response Process
- A maintainer will acknowledge receipt within 3 business days.
- Impact will be assessed and a fix will be planned or scheduled.
- Once a fix is available, a patched release will be issued with notes in `CHANGELOG.md`.
- You will be credited for the report unless you request otherwise.
